# TESTS.md

## 1. Цель тестирования

Проверить работоспособность сервиса голосового управления умным домом в двух плоскостях:
- техническая надежность (корректность команд, устойчивость, задержки);
- пользовательское восприятие (понятность интерфейса, удобство, качество ответа).

Тестирование проводилось на итоговой сборке приложения с локальным шлюзом и Home Assistant.

## 2. Контур тестирования

- Мобильный клиент Android (ASR + TTS).
- SmartHome Gateway (LAN, parser mode: `rules`, `llm_safe`, `llm`).
- Локальная модель (single-model режим).
- Home Assistant + Zigbee-лампа `light.lampa1`.

## 3. Техническое тестирование

### 3.1 Набор проверок

1. Базовые команды управления светом:
- `включи свет`;
- `выключи свет`;
- `сделай потише`;
- `сделай поярче`.

2. Команды цвета:
- `сделай белый свет`;
- `включи зеленый свет`;
- `синий свет`.

3. Проверка уточнений:
- команда без комнаты при отсутствии контекста;
- выбор комнаты из предложенного списка.

4. Проверка сетевой устойчивости:
- повторная отправка после кратковременного обрыва связи;
- таймауты и обработка ошибок в UI.

5. Производительность:
- измерение времени ответа в режиме только одной модели (`llm`).

### 3.2 Результаты технических тестов

| Группа тестов | Ожидаемый результат | Фактический результат | Статус |
| --- | --- | --- | --- |
| Базовые команды света | Корректный вызов команд HA | Команды исполняются корректно | Пройдено |
| Команды цвета | Корректная интерпретация цветов | После доработки цвета распознаются верно | Пройдено |
| Уточнение комнаты | Запрос уточнения при нехватке контекста | Приложение задает уточнение и выполняет команду | Пройдено |
| Сеть и ошибки | Понятное сообщение о проблеме | Ошибки отображаются, повтор работает | Пройдено |
| Время ответа (`llm`) | Снижение задержки | Улучшено с ~80 c до ~30 c | Пройдено |

## 4. Пользовательское тестирование (фокус-группа)

### 4.1 Состав фокус-группы

- 6 пользователей (без подготовки по ML/IoT).
- У каждого был сценарий из 8-10 команд (голос и текст).

### 4.2 Сценарии

1. Включение/выключение света.
2. Изменение яркости.
3. Изменение цвета лампы.
4. Команды без комнаты с последующим уточнением.
5. Просмотр журнала и диагностической информации.

### 4.3 Обратная связь

- Пользователи ожидают голосовой ответ после выполнения команды.
- Важна стабильность распознавания речи в бытовых условиях.
- Критично заметно время ожидания в `llm` режиме.
- Команды цвета должны пониматься без дополнительной формулировки.

### 4.4 Результаты пользовательского теста

| Критерий | До улучшений | После улучшений |
| --- | --- | --- |
| Удобство голосового взаимодействия | Среднее | Высокое (за счет ASR+TTS) |
| Понятность ответа системы | Средняя | Высокая (голос + текст) |
| Ожидание ответа в `llm` | Низкая удовлетворенность | Существенно лучше (80 c -> 30 c) |
| Команды цвета | Частые ошибки | Стабильное распознавание |

## 5. Итог

Тестирование показало, что сервис выполняет целевой сценарий локального управления умным домом, а также устойчиво работает в пользовательских условиях. Ключевые проблемы, выявленные на тестах (скорость ответа, команды цвета, голосовой цикл), устранены в следующей итерации.
